StudyTime AI Hosting Instructions

1. Ensure Backend is Functional
- Python 3.11+ is required (3.13 OK).
- Dependencies:
  cd backend
  pip install -r requirements.txt
- Add backend/.env:
  MONGO_URL="mongodb://localhost:27017"
  DB_NAME="studytime_db"
  CORS_ORIGINS="*"

2. Run Backend Locally
- Run:
  cd backend
  python server.py
- Check: http://localhost:8001/api/chat/<session_id> should respond (or 404 if no session yet).

3. Frontend Setup Locally
- Node 18+ required.
  cd frontend
  npm install --legacy-peer-deps
- Add frontend/.env:
  VITE_BACKEND_URL=http://localhost:8001

4. Run Frontend Locally
- Run:
  cd frontend
  npm run dev
- Check: http://localhost:3000 loads app.

5. Save Gemini API Key in App
- In app, click settings icon (⚙️).
- Paste key (model gemini-2.5-flash recommended).
- Save.

6. Use App
- Upload PDF or paste text
- Click Analyze
- Review tabs: Summary, MindMap, Chat, Quiz
- New session created after each analyze.

7. Session Management (New)
- Sessions stored in localStorage key `study_sessions`.
- Each session includes: id, filename/title, text, studyData, createdAt.
- Navbar session dropdown selects active session.
- Delete button removes selected session and returns to upload if none.

8. Railway Backend Hosting (Recommended)
- Create a project in Railway from GitHub.
- Select backend directory.
- Set Railway environment variables:
  MONGO_URL
  DB_NAME
  CORS_ORIGINS
- Deploy the backend and copy the generated URL (e.g., https://studytime-backend.railway.app).

9. Netlify Frontend Hosting
- Create a site from your GitHub repo in Netlify.
- Base directory: frontend.
- Build command: npm run build
- Publish directory: dist
- Set environment variable:
  VITE_BACKEND_URL=https://<your-railway-backend-url>
- Deploy and test.

10. Production smoke tests
- Open Netlify frontend URL.
- Validate upload + analyze flows.
- Validate session dropdown and scoped tab data.
- Validate API key and analysis works.
